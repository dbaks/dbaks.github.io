自动生成的构造函数

发表于 2021-05-12 | 分类于 CPP
  • c++编译器在某些特定的情况下,自动为类生成一个无参的构造函数(不是说类没有构造函数就会生成)

[ PS: ] 很多书上都说 : 编译器会为每一个类生成无参的空的构造函数 (这对吗? ,这不对,只有特定情况下会生成)

解释 : class Person92 中并未自定义构造函数.可以看到,运行时汇编中并没有调用构造函数,说明此时并不会生成构造函数

一些不会生成的情况(想到的)

  • new 对象时一定要调用构造函数,但是如果不存在,编译器只会报错,而不是自动生成一个

  • 特定的会生成自动生成构造函数的情况(当Person对象生成后需要做一些后续动作时)

    • 成员变量在生成时初始化(下图为某个成员变量存在初始化值)

    • 有成员函数为虚函数Virtual ,创建的对象会生成四个字节来生成指向虚表地址的指针(其实和成员变量存在初始值一样,需要在生成对象的时候对内存赋值)

    • 虚继承其他类,也是虚表地址的关系

    • 创建A对象时,A对象内部存在B类对象,并且B类对象存在构造函数或者以上情形

    • 父类存在构造函数或者存在以上情形

      • 总结: 当对象生成后,如果需要进行额外的内存操作或者函数调用,编译器就会自动创建无参的构造函数

匿名对象(临时对象)

发表于 2021-05-11 | 分类于 CPP

有名字的,有指针指着的不是匿名对象

匿名对象: 顾名思义是没有名字的对象

生存周期: 一句语句( 在当前语句创建匿名对象,在下一句语句执行前会 销毁/执行析构函数 )

扩展:

  • 当匿名对象作为函数参数时,编译器会对其做出优化,与传入非匿名对象不同
  • 当匿名对象最为函数返回值时,也会优化,相当于直接放在main中一样

对象类型参数和返回值

发表于 2021-05-11 | 分类于 CPP
  • 存在的问题:

    [对象类型作为参数]可能会产生一些不必要的中间值.

    • 对象作为参数传入时,会调用拷贝构造函数,建立一个新的对象(中间对象)
      [ note ] 这和一般函数是一样的,所以一般会使用引用,而不是直接传参

      [对象作为返回值] 会产生中间值

      • 原因 : 函数栈空间在函数结束调用之后就会释放,所以需要将函数栈空间中的对象在函数结束前提前将其复制到main函数的栈空间中,因此会调用拷贝构造函数 (如果是在main中调用该函数)
    • 实现方法(底层) : 在调用 test2() 时,传入一个指针(指向main栈空间的指针),当对象返回时,调用拷贝构造函数将其拷贝到main函数的栈空间中
