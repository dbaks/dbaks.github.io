两个不同类互相调用问题

发表于 2021-05-14

起初碰到这个问题是在学习友元的时候,

为了方便说明我把Math类称为A类,将Point_friend_93成为B类,起只是在两个类的头文件中简单#include了对方的头文件

结果发现,在B类中声明A类的函数为友元时,编译器报错,(说他找不到A中的那个函数,认为我们没有定义这个函数)

具体过程…我也还没搞清楚,但是找到了解决方案,

1.只在A类的头文件中引入B类头文件

2.由于B类中没有#include “A.h”,所以此时B类无法使用A类,所以要在B类头文件的开头声明一下A类 (class A;)

3.然后在B类的cpp文件中#include A.h

此处因该涉及到 : 向前声明,不完全类型,单独编译,以及类的基础知识详情可见此篇博客

内部类

发表于 2021-05-14

内部类

定义: 定义在一个类内部的类叫做内部类(嵌套类)

特点:

  • 可以用访问控制符控制访问权限

  • 内部类成员函数可以访问外部类的所有成员

  • 外部类无法为访问内部类成员所有成员

  • 内部类可以不带类名或对象名直接访问Static成员

  • 不会影响外部类的内存布局,只影响内部类的访问权限

  • 内部类声明和实现可分离

局部类

定义: 在函数内部定义的类

特点:

  • 局部类的声明和实现必须放在类里
  • 不支持static成员变量(static成员变量需要在类外初始化)
  • 成员函数不可访问函数的局部变量,但是可以访问函数里的static变量
  • 可以访问全局变量

自动生成的构造函数

发表于 2021-05-12 | 分类于 CPP
  • c++编译器在某些特定的情况下,自动为类生成一个无参的构造函数(不是说类没有构造函数就会生成)

[ PS: ] 很多书上都说 : 编译器会为每一个类生成无参的空的构造函数 (这对吗? ,这不对,只有特定情况下会生成)

解释 : class Person92 中并未自定义构造函数.可以看到,运行时汇编中并没有调用构造函数,说明此时并不会生成构造函数

一些不会生成的情况(想到的)

  • new 对象时一定要调用构造函数,但是如果不存在,编译器只会报错,而不是自动生成一个

  • 特定的会生成自动生成构造函数的情况(当Person对象生成后需要做一些后续动作时)

    • 成员变量在生成时初始化(下图为某个成员变量存在初始化值)

    • 有成员函数为虚函数Virtual ,创建的对象会生成四个字节来生成指向虚表地址的指针(其实和成员变量存在初始值一样,需要在生成对象的时候对内存赋值)

    • 虚继承其他类,也是虚表地址的关系

    • 创建A对象时,A对象内部存在B类对象,并且B类对象存在构造函数或者以上情形

    • 父类存在构造函数或者存在以上情形

      • 总结: 当对象生成后,如果需要进行额外的内存操作或者函数调用,编译器就会自动创建无参的构造函数
